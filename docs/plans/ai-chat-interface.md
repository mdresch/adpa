# Implementation Plan: AI Chat Interface

## Goal Description
Implement an intelligent AI Chat Interface embedded within the document viewer to allow users to ask context-aware questions about the document, request inline refinements (e.g., "Make section 3 more technical"), and proactively receive suggested improvements. This interface transitions the application from a one-shot generation tool into an interactive, iterative drafting environment.

## Proposed Changes

### `app/documents/[id]/chat/page.tsx`
- [NEW] Build the Chat UI sliding panel in the frontend.
- [NEW] Implement `socket.io-client` for real-time bidirectional streaming of AI responses.
- [NEW] Create actionable message components (e.g., allowing users to click "[Apply Changes]" or "[Preview]" on AI-suggested content diffs).

### `server/src/modules/intelligence/ChatService.ts`
- [NEW] Build the AI backend service that receives `ChatMessage` payloads.
- [NEW] Inject the full generated document as context into the LLM's system prompt to enable "Context-Aware" Q&A.
- [NEW] If the user requests a modification, the service must output structured JSON representing an `action` payload (`apply_suggestion` or `regenerate_section`) alongside conversational text.

### `server/src/modules/documents/DocumentModificationController.ts`
- [NEW] Create an API endpoint to receive and apply the `action` payloads generated by the ChatService directly to the database document record.

## Verification Plan

### Automated Tests
- Unit test: `ChatService` to ensure it successfully parses user intent (e.g., distinguishing between a Q&A question vs an editing command) and triggers the correct structured output.
- Integration test: Ensure applying an `apply_suggestion` payload correctly modifies the target section in the database and creates a new version history snapshot.

### Manual Verification
- Open a generated document and open the Chat Interface.
- Ask: "What stakeholders are mentioned in this document?" and verify the AI accurately answers based on the document's content.
- Ask: "Make the executive summary more concise." 
- Verify the AI returns a shortened draft with an `[Apply Changes]` button.
- Click `[Apply Changes]` and verify the executive summary in the main document pane immediately updates.
