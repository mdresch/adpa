# Implementation Plan: DOCX Export

## Goal Description
Implement native Microsoft Word (`.docx`) export functionality that preserves styling, structure, and formatting. Unlike PDF export which creates immutable documents, DOCX export requires creating a file that remains highly editable by human stakeholders while maintaining the professional branding, syntax highlighted code blocks, and native Word tables generated by the platform.

## Proposed Changes

### `server/src/modules/export/DOCXExportService.ts`
- [NEW] Implement the core DOCX generation engine utilizing the `docx` library (or an AST-to-Word transformer).
- [NEW] Build the parser that transforms the `Document` Markdown structure directly into native Word Elements (`Paragraph`, `Table`, `HeadingLevel`).

### `server/src/modules/export/WordStyler.ts`
- [NEW] Create a styling engine that maps CSS-like styles (from the user's `styling` payload) to native Word styles.
- [NEW] Implement specialized rendering for complex structures:
  - Tables: Ensure column widths are properly calculated and grid/striped styling is applied natively.
  - Code Blocks: Map monospace fonts with shaded background highlighting.
  - Bullet Lists: Map to native Word list styles with accurate indentation.

### `server/src/modules/documents/ExportController.ts`
- [MODIFY] Add `POST /api/documents/:id/export/docx` endpoint.
- [MODIFY] Validate the incoming `DOCXExportOptions` payload, which includes metadata like author, company, and styling preferences.

## Verification Plan

### Automated Tests
- Unit test: `WordStyler` to ensure Markdown AST nodes (like `heading`, `table`, `code`) are successfully converted into `docx` package objects without throwing errors.
- Integration test: Generate a full `.docx` buffer in-memory and assert its validity and size.

### Manual Verification
- Generate a 50-page complex document (e.g., Technical Architecture with code blocks, tables, and images).
- Trigger the DOCX Export via the UI.
- Open the resulting file in Microsoft Word and LibreOffice.
- Verify that Tables are native (editable, not images), code blocks are properly shaded, and the Table of Contents correctly links to headings.
