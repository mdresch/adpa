## 📁 **Where to Access**

### **In the Project Library:**

1. Navigate to **Projects** → Select your project
2. Go to **Documents** tab
3. Find document: `"Template Name - MM/DD/YYYY"`
4. Click to view:
   - **Content**: Full markdown document
   - **Metadata**: All pipeline metrics (visible in document details)

### **Metadata Fields Visible:**

| Field | Description | Example |
|-------|-------------|---------|
| **Template** | Template name & version | "PMBOK Stakeholder Plan v1" |
| **Framework** | Methodology framework | "PMBOK 7" |
| **Generated By** | User who ran pipeline | "Admin User" |
| **Word Count** | Total words | 1,247 |
| **File Size** | Size in KB | "7.67 KB" |
| **Processing Time** | Total duration | "58.71s" |
| **AI Model** | Provider & model used | "google/gemini-2.5-flash" |
| **Tokens Used** | AI tokens consumed | 2,547 |
| **Generation Cost** | Estimated cost | "$0.000025" |
| **Quality Score** | Overall quality | "70.91%" |
| **File Hash** | SHA-256 checksum | "a1b2c3d4..." |
| **Compression** | Compression ratio | 0.95 |
| **Language** | Document language | "en" |
| **MIME Type** | Content type | "text/markdown" |

## 🔍 **Metadata Use Cases**

### **1. Quality Tracking**
- Compare quality scores across documents
- Identify which templates produce highest quality
- Track improvement over time

### **2. Cost Analysis**
- Monitor AI token usage
- Calculate generation costs
- Optimize provider selection for cost

### **3. Performance Monitoring**
- Track processing times per stage
- Identify bottlenecks
- Optimize slow stages

### **4. Audit Trail**
- Who generated the document
- When it was created
- Which AI model/provider was used
- Complete processing history

### **5. Reproducibility**
- Template version used
- Framework followed
- Processing configuration
- Exact pipeline settings

### **6. Compliance**
- File integrity (hash verification)
- Quality assurance metrics
- Assessment results
- Issue tracking
