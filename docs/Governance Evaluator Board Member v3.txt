Governance Evaluator Board Member
Audits compliance and rules enforcement against stated architecture frameworks.
Findings
**Exceptional BABOK V3 Strategy Analysis Alignment:** The template guidance demonstrates outstanding alignment with BABOK v3's Strategy Analysis knowledge area. It explicitly enforces 'Determine Value' through quantified benefits and NPV, 'Define Transition Requirements,' and implicitly covers 'Analyze Current State' (via root cause analysis) and 'Define Future State' (via success criteria). This ensures a robust strategic justification for the business case.
**Robust Financial and Analytical Rigor:** The template mandates critical analytical techniques including Root Cause Analysis, Net Present Value (NPV) calculation, comprehensive Cost-Benefit Analysis, and Sensitivity Analysis. This level of rigor is essential for building a 'bulletproof' investment proposal and aligns with best practices for financial justification.
**Emphasis on Measurable Success Criteria and Traceability:** The guidance requires success criteria to be measurable, include baseline metrics, and align with the initial business need, with specified validation methods. This directly supports BABOK's focus on defining clear, verifiable requirements and value realization.
**Professional Synthesis and Communication Focus:** The template explicitly requires professional, executive-level language and adherence to specific naming conventions (e.g., 'PMBOK Guide' vs. 'PMBOK'), promoting clear and effective communication of complex information to stakeholders, which is crucial for decision-making.
**Partial PMBOK Governance Integration:** While the template's persona operates at the intersection of BABOK and PMBOK (Value Delivery), and focuses on 'Outcomes over Outputs,' it does not explicitly mandate several key PMBOK-specific governance elements outlined in the audit instructions.
Compliance Gaps

Remediation Guidelines
**Integrate PMBOK Governance Elements:** To fully meet the implications of mentioning PMBOK and to strengthen the overall governance aspect, consider adding prompts within the guidance or the generated document for: 1) Defining key project roles (e.g., Sponsor, Project Manager, Key Stakeholders), 2) Briefly discussing the anticipated project lifecycle (Predictive, Adaptive, Hybrid), 3) Initial considerations for risk escalation paths, and 4) A basic outline for communication cadence relevant to the project's initiation.
**Guidance for Discount Rate Determination:** For enhanced financial rigor in NPV calculations, provide guidance on how the discount rate 'r' should be determined (e.g., Weighted Average Cost of Capital, company-specific hurdle rate, or a default assumption if none is provided).
**Explicitly List BABOK Knowledge Areas/Tasks (Minor):** While many BABOK concepts are well-integrated, a small refinement could be to explicitly list the primary BABOK Knowledge Areas and/or Tasks addressed (e.g., 'Strategy Analysis: Tasks 7.1, 7.2, 7.3, 7.5; Requirements Analysis and Design Definition: Task 7.4') within the 'Core Competencies & BABOK V3 Alignment' section to further highlight compliance.
Counterfactual Challenger Board Member
Stresses template instructions, loops, and vagueness to prevent AI hallucinations.
Findings
The prompt implicitly assumes the user will provide comprehensive 'raw project data' including all specific financial figures (cash flows, discount rate, initial investment, detailed costs, benefits) and multiple solution options required for robust NPV, CBA, and comparative analysis. This is a critical input dependency not explicitly addressed.
The instruction to ensure 'all benefits... are expressed as projected monetary values' creates a loophole for non-monetary benefits. The AI might either ignore them, invent arbitrary monetary values, or fail to produce a comprehensive picture if true quantification is not possible from user input.
The depth and specific format for certain analyses (e.g., Root Cause Analysis using '5 Whys or Fishbone logic', 'Transition State' discussion, 'Sensitivity Analysis') are vague, potentially leading to superficial or inconsistent output quality.
The prompt mentions 'ROI, NPV' as measurable returns but only provides a formula for NPV, leaving the methodology for ROI calculation unspecified.
The requirement for 'Solution Integrity' to articulate differences between options (Pros/Cons/Costs) implies multiple options will be provided by the user. If only one solution is given, the AI will be forced to hallucinate alternative solutions, compromising the objectivity of the analysis.

Prompt Sharpening Guidelines
**Explicit Data Requirements & Handling:** Add a section explicitly detailing the expected user inputs for financial data, project context, and solution options. Crucially, instruct the AI on how to handle *missing* data: 'If specific financial figures (e.g., cash flows, discount rate, initial investment, multiple options) are not explicitly provided by the user, you MUST either: a) Request this information from the user before proceeding, OR b) Clearly state in your output that certain assumptions were made due to missing data, detailing each assumption (e.g., 'Assuming an initial investment of X based on industry averages...'). Do NOT invent detailed, arbitrary numbers without explicit disclosure.'
**Define Analysis Depth & Format:** For 'Root Cause Analysis,' clarify: 'Apply 5 Whys/Fishbone logic in your narrative to identify root causes; do not explicitly detail each 'Why' step or diagram unless specifically requested by the user.' For 'Transition State' and 'Sensitivity Analysis,' specify expected detail, e.g., 'Provide a high-level overview of key transition phases with estimated timelines' and 'Recalculate key metrics (e.g., NPV) under the sensitivity scenarios.'
**ROI Calculation Guidance:** Provide a simplified formula for ROI or clarify its status: 'For ROI, use the formula: ROI = ((Total Benefits - Total Costs) / Total Costs) * 100%. If data is insufficient, state this clearly.'
**Handling Non-Monetary Benefits:** Modify the 'Quantified Value' instruction: 'For benefits that cannot be credibly quantified monetarily based on user input, clearly state them qualitatively (e.g., Improved Employee Morale, Enhanced Regulatory Compliance) and briefly explain the challenges in their quantification, rather than forcing an arbitrary monetary value.'
**Handling Single Solution Input:** For 'Solution Integrity,' instruct: 'If the user provides only one proposed solution, you may either (a) generate 1-2 plausible, high-level alternative solutions for comparative analysis (explicitly labeling them as AI-generated alternatives) or (b) state that comparative analysis is limited due to a single provided option.'
**Clarify 'Current Dates':** Refine 'All dates must be current' to: 'Use the current generation date for report/analysis dates. For project timelines and future milestones, use relative timeframes (e.g., 'Q3 2024', 'FY 2025') unless specific calendar dates are provided by the user.'
