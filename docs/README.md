## 📝 Documentation Standards

When adding new documentation:

1. **Place in appropriate category** - Use the numbered folders for easy navigation
2. **Use clear naming** - Descriptive file names in UPPER_CASE.md format
3. **Include metadata** - Date, version, author information at the top
4. **Cross-reference** - Link to related documentation
5. **Keep updated** - Mark superseded docs and move to archive
