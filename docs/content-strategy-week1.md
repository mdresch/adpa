# Content Strategy - Week 1

**Document Version:** 1.0  
**Date:** Week 1, Phase 1  
**Status:** Content Strategy & Production Plan

---

## Executive Summary

This document defines the content strategy for ADPA's maturity portal, including detailed outlines for 5 maturity levels, a 12-week content production calendar, and frontmatter conventions for all content types.

---

## 1. Content Strategy Overview

### 1.1 Purpose

The content strategy ensures:
- **Comprehensive Coverage:** All maturity levels and knowledge areas covered
- **Consistent Quality:** Standardized content structure and format
- **Progressive Learning:** Content supports users at every maturity level
- **SEO Optimization:** Content optimized for search and discovery
- **Scalability:** Clear process for ongoing content creation

### 1.2 Content Types

1. **Maturity Level Guides** - Deep dives into each of 5 maturity levels
2. **Knowledge Area Guides** - PMBOK 10 knowledge areas
3. **Quick Start Guides** - Short, actionable guides (500 words)
4. **Templates** - Reusable document templates
5. **Case Studies** - Real-world examples and success stories
6. **Best Practices** - Proven methods and standards
7. **Video Scripts** - Scripts for video tutorials
8. **Webinar Content** - Training and educational content

### 1.3 Content Principles

1. **User-Centric:** Content addresses real user needs
2. **Actionable:** Every piece includes actionable steps
3. **Evidence-Based:** Content backed by PMBOK/BABOK/DMBOK standards
4. **Progressive:** Content builds from beginner to expert
5. **Traceable:** All content links to source frameworks
6. **SEO-Optimized:** Content optimized for search discovery

## 2. Maturity Level Content Outlines

### 2.1 Level 1: Initial / Ad Hoc

**Target Word Count:** 2,000-3,000 words  
**Target Audience:** Organizations just starting PM journey  
**Content Goal:** Help users recognize their current state and take first steps

#### Content Structure

**1. Overview Section (400-500 words)**
- **Definition:** What Level 1 means in practical terms
- **Characteristics:**
  - Unpredictable processes
  - Success depends on heroes
  - Firefighting mode
  - Minimal documentation
  - No standard templates or processes
  - Reactive approach to project management
- **Organizational Symptoms:**
  - Projects succeed or fail unpredictably
  - No consistent methodology
  - Knowledge exists only in individuals' heads
  - High dependency on key personnel
- **Success Indicators:** How to recognize you're at Level 1

**2. Assessment Criteria (300-400 words)**
- **Self-Assessment Questions:**
  - Do you have standard project templates?
  - Is project success predictable?
  - Do you track project metrics consistently?
  - Are processes documented?
- **Document Indicators:**
  - Inconsistent document formats
  - Missing key project documents
  - No document templates
  - Documents created ad-hoc
- **Process Maturity Signs:**
  - No defined project lifecycle
  - No standard approval processes
  - No consistent reporting
- **Red Flags:**
  - Projects frequently over budget/schedule
  - High team turnover
  - Constant firefighting

**3. Challenges & Pain Points (400-500 words)**
- **Common Problems:**
  - Unpredictable project outcomes
  - Knowledge loss when key people leave
  - Inability to replicate success
  - Constant crisis management
  - No learning from past projects
- **Why Organizations Get Stuck:**
  - Lack of awareness of current state
  - No clear improvement path
  - Resource constraints
  - Resistance to change
- **Risks and Consequences:**
  - Project failures
  - Wasted resources
  - Team burnout
  - Lost opportunities
- **Real-World Examples:** Case studies of Level 1 organizations

**4. Improvement Path to Level 2 (600-800 words)**
- **Quick Wins (0-3 months):**
  - Create basic project charter template
  - Establish weekly status meetings
  - Start tracking project costs and schedules
  - Document lessons learned after each project
- **Strategic Initiatives (3-6 months):**
  - Develop 5-10 core project templates
  - Define basic project lifecycle
  - Establish change request process
  - Create risk register template
- **Timeline Expectations:**
  - Quick wins: 1-3 months
  - Full transition: 6-12 months
- **Resource Requirements:**
  - 0.5 FTE Project Manager
  - Budget: $10K-$20K
  - Training: 16 hours basic PM training
- **Success Metrics:**
  - Templates created and in use
  - Consistent project documentation
  - Reduced project failures
  - Improved team confidence

**5. Success Stories (300-400 words)**
- **Case Study 1:** Small organization (10-50 employees)
  - Starting point
  - Actions taken
  - Results achieved
  - Timeline
- **Case Study 2:** Medium organization (50-200 employees)
  - Different challenges
  - Approach taken
  - Outcomes
- **Lessons Learned:**
  - What worked well
  - What didn't work
  - Key takeaways

**6. Tools & Resources (300-400 words)**
- **Templates for Level 1:**
  - Basic Project Charter
  - Simple Project Plan
  - Risk Register
  - Status Report Template
  - Lessons Learned Template
- **Recommended Tools:**
  - Basic project management software
  - Document templates library
  - Simple tracking spreadsheets
- **Training Materials:**
  - PM fundamentals course
  - Template usage guides
  - Best practices library
- **External Resources:**
  - PMBOK Guide references
  - Industry articles
  - Community forums

## 7. Frontmatter Convention

### 7.1 Reference Document

**See:** `docs/mdx-frontmatter-spec.md` for complete frontmatter specification.

### 7.2 Quick Reference

**Required Fields:**
- `title` - Document title
- `slug` - URL-friendly identifier

**Recommended Fields:**
- `area` - Content area (getting-started, features, etc.)
- `level` - Expertise level (beginner, intermediate, advanced, expert)
- `framework` - Framework alignment (PMBOK, BABOK, DMBOK)
- `tags` - Multi-dimensional tags
- `related` - Related document slugs
- `readingTime` - Estimated reading time in minutes
- `seo` - SEO metadata (title, description, keywords)
- `status` - Publication status (draft, review, published, archived)
- `author` - Author name
- `created` - Creation date
- `updated` - Last update date

### 7.3 Content-Specific Frontmatter

**Maturity Level Guides:**
```yaml
area: "features"
level: "beginner" # Level 1-2: beginner, Level 3: intermediate, Level 4-5: advanced/expert
framework: "PMBOK"
tags: ["maturity-levels", "level-{N}", "pmbok"]
```

**Knowledge Area Guides:**
```yaml
area: "features"
level: "intermediate"
framework: "PMBOK"
tags: ["knowledge-areas", "{area-name}", "pmbok"]
```

**Quick Start Guides:**
```yaml
area: "getting-started"
level: "beginner"
tags: ["quick-start", "{topic}", "getting-started"]
```

**Templates:**
```yaml
area: "features"
level: "beginner" # or appropriate level
framework: "PMBOK" # or appropriate framework
tags: ["templates", "{template-type}", "{framework}"]
```

## 8. Content Quality Standards

### 8.1 Writing Guidelines

1. **Clarity:** Use clear, concise language
2. **Actionability:** Every piece includes actionable steps
3. **Examples:** Include real-world examples
4. **Visuals:** Use diagrams, tables, and lists
5. **Consistency:** Follow established style guide
6. **Accuracy:** Verify all information against PMBOK/BABOK/DMBOK

### 8.2 SEO Guidelines

1. **Keywords:** Include relevant keywords naturally
2. **Headings:** Use descriptive H2/H3 headings
3. **Meta Descriptions:** Compelling 150-160 character descriptions
4. **Internal Links:** Link to related content
5. **External Links:** Link to authoritative sources
6. **Alt Text:** Descriptive alt text for images

### 8.3 Review Process

1. **Content Creation:** Author creates draft
2. **Self-Review:** Author reviews against checklist
3. **Peer Review:** Another team member reviews
4. **Technical Review:** PMBOK/BABOK/DMBOK accuracy check
5. **SEO Review:** SEO optimization check
6. **Final Approval:** Content lead approves
7. **Publication:** Content published with proper frontmatter

6. **SEO:** SEO specialist optimizes
7. **Design:** Designer creates visuals
8. **Approval:** Content lead approves
9. **Publication:** Content published with frontmatter
