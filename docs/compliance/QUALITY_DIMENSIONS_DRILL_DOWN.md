# Quality Dimensions Drill-Down Feature

**Last Updated**: 2026-01-24  
**Status**: ✅ Implemented - Drill-Down Details Available for All Quality Dimensions  
**Purpose**: Enable users to see detailed breakdowns of how each quality dimension score is calculated

---

## Overview

The Quality Audit Report now includes **drill-down functionality** for each quality dimension in the **Scores** tab, providing users with:

1. **Detailed Score Breakdown** - How each percentage is calculated
2. **Scoring Factors** - What factors were checked and their point values
3. **Content Analysis** - Specific details about document structure and content
4. **Contribution to Overall** - How each dimension contributes to the overall quality score
5. **Improvement Recommendations** - Specific actions to improve scores

---

## User Interface

### Location: Quality Audit Modal → Scores Tab

**Access Path:**
1. Navigate to Document Metadata page
2. Click "View Full Report" button (or "Run Quality Audit" if not yet run)
3. Click on "Scores" tab
4. Click on any quality dimension card to expand details

---

## Features

### 1. **Expandable Quality Dimension Cards**

Each quality dimension card is now **clickable** and expands to show:
- ✅/❌ Status indicators for each scoring factor
- Points earned vs. maximum points
- Detailed analysis of document structure and content
- Calculation formula
- Contribution to overall quality score
- Improvement recommendations

### 2. **Overall Quality Score Breakdown**

The overall quality score card includes:
- **"How is this calculated?"** expandable section
- Individual dimension contributions with weights
- Calculation formula showing weighted average
- Visual breakdown of each dimension's contribution

**Example:**
```
Completeness: 100% × 20% = 20.0%
Consistency: 85% × 15% = 12.8%
Professional Quality: 90% × 20% = 18.0%
Standards Compliance: 75% × 20% = 15.0%
Accuracy: 80% × 15% = 12.0%
Context Relevance: 70% × 10% = 7.0%

Total: 84.8% (rounded to 85%)
```

## Quality Dimensions with Drill-Down

### 1. **Completeness** (0-100%, Weight: 20%)

**Scoring Factors:**
- **Main Title** (25 points): Document starts with a main title (H1 heading)
- **Section Headers** (25 points): Document has 3+ main sections (H2 headings)
- **Tables** (25 points): Document includes structured data in tables (10+ table cells)
- **Lists** (25 points): Document includes bulleted or numbered lists (5+ list items)

**Calculation:**
```
Score = 25 (Title) + 25 (Headers) + 25 (Tables) + 25 (Lists) = 100%
```

**Example Breakdown:**
- ✓ Main Title: 25 / 25 pts - Main title found
- ✓ Section Headers: 25 / 25 pts - 8 sections found
- ✓ Tables: 25 / 25 pts - 45 table cells found
- ✓ Lists: 25 / 25 pts - 12 list items found

**Recommendations:**
- Add a main title at the beginning of the document
- Add more section headers (currently X, target: 3+)
- Add tables to organize structured data
- Add bulleted or numbered lists for better readability

---

### 2. **Consistency** (0-100%, Weight: 15%)

**Scoring Factors:**
- **Table of Contents Structure** (20 points): Document has headings (H1/H2/H3) for auto-generated ToC (ToC is automatically generated by the document viewer from headings)
- **Consistent Header Structure** (25 points): Document has 1-15 main sections (balanced structure)
- **Sentence Flow** (30 points): Average sentence length 10-25 words
- **Uniform Section Length** (25 points): Sections are similar in length (~200 words each)

**Calculation:**
```
Score = 20 (TOC Structure) + 25 (Headers) + 30 (Flow) + 25 (Uniform) = 100%
```

**Example Breakdown:**
- ✗ Table of Contents: 0 / 20 pts - Add headings (H1/H2/H3) to enable auto-generated table of contents
- ✓ Consistent Header Structure: 25 / 25 pts - 8 sections (good balance)
- ✓ Sentence Flow: 30 / 30 pts - Good sentence flow (18.5 words)
- ⚠ Uniform Section Length: 10 / 25 pts - Section length varies (150 words avg, target: ~200)

**Recommendations:**
- Add headings (H1/H2/H3) to enable auto-generated table of contents in the document viewer
- Balance section count (currently X, target: 1-15)
- Adjust sentence length (currently X words, target: 10-25)
- Balance section lengths (currently X words avg, target: ~200)

### 6. **Context Relevance** (0-100%, Weight: 10%)

**Scoring Factors:**
- **Project Context** (35 points): Document references project context (3+ keywords)
- **Framework Alignment** (25 points): Document aligns with framework (PMBOK, BABOK, DMBOK)
- **Actionable Content** (25 points): Document includes actionable statements (should, must, will, shall)
- **Framework Keywords** (15 points): Framework keywords found (5+ keywords)

**Calculation:**
```
Score = 35 (Context) + 25 (Framework) + 25 (Actionable) + 15 (Keywords) = 100%
```

**Keywords Checked:**
- `project`, `stakeholder`, `risk`, `scope`, `budget`, `schedule`, `quality`, `resource`

**Example Breakdown:**
- ✓ Project Context: 35 / 35 pts - Strong project context (7 keywords)
- ✓ Framework Alignment: 25 / 25 pts - Framework alignment found
- ✓ Actionable Content: 25 / 25 pts - Actionable content found
- ✓ Framework Keywords: 15 / 15 pts - Excellent keyword coverage (7 keywords)

**Recommendations:**
- Add more project context keywords (currently X, target: 3+)
- Reference framework (PMBOK, BABOK, or DMBOK)
- Add actionable statements (should, must, will, shall)
- Add more framework keywords (currently X, target: 5+)

## Visual Features

### Expandable Cards
- **Collapsed State:** Shows dimension name, description, score, weight, and contribution
- **Expanded State:** Shows detailed breakdown with all factors, calculations, and recommendations
- **Chevron Icon:** Indicates expandable state (down = collapsed, up = expanded)

### Color Coding
- **Green (✓)**: Factor passed
- **Red (✗)**: Factor failed
- **Yellow (⚠)**: Factor partially met
- **Progress Bars**: Visual representation of score vs. maximum

### Status Indicators
- **CheckCircle (✓)**: Factor passed
- **XCircle (✗)**: Factor failed
- **Details Text**: Specific measurements and targets

## Technical Implementation

### Frontend Component: `QualityAuditModal.tsx`

**Enhanced Components:**
- `QualityMetricCard` - Now expandable with drill-down details
- `QualityDimensionBreakdownDetails` - Shows detailed breakdown
- `OverallQualityBreakdown` - Shows overall score calculation
- `calculateQualityDimensionBreakdown` - Calculates detailed breakdown on-demand

**Data Flow:**
1. User clicks on quality dimension card
2. Component fetches document content (if not already loaded)
3. `calculateQualityDimensionBreakdown()` analyzes document content
4. Breakdown details displayed in expanded card

### Backend Enhancement: `qualityAuditService.ts`

**Updated Method:**
- `getDocumentAudit(documentId, includeContent)` - Now accepts `includeContent` parameter
- When `includeContent=true`, includes document content in response
- Includes `framework_used` from document for context analysis

**API Endpoint:**
- `GET /api/quality-audits/document/:documentId?includeContent=true`
- Optional query parameter to include document content for breakdown analysis

## Usage Examples
