## Quick Start Paths

### Path 1: New User (Week 1)

**Day 1** (15 minutes):
1. ✅ Read [Getting Started Guide](GETTING_STARTED_GUIDE.md) - Sections 1-4
2. ✅ Log in and explore dashboard
3. ✅ Create your first project
4. ✅ Generate your first document

**Day 2-3** (30 minutes):
1. ✅ Read Getting Started Guide - Sections 5-8
2. ✅ Try different document templates
3. ✅ Export a document to PDF
4. ✅ Upload an existing document

**Day 4-5** (30 minutes):
1. ✅ Review [Quick Reference Guide](QUICK_REFERENCE_GUIDE.md)
2. ✅ Learn keyboard shortcuts
3. ✅ Set up an integration (optional)
4. ✅ Explore analytics dashboard

**By End of Week 1**: You should be comfortable creating projects, generating documents, and using core features.

---

### Path 2: Portfolio Manager (Week 1-2)

**Week 1** (1 hour):
1. ✅ Complete New User path above
2. ✅ Create your first program
3. ✅ Add projects to the program
4. ✅ Explore program dashboard

**Week 2** (2 hours):
1. ✅ Read [Portfolio Prioritization Matrix Guide](PORTFOLIO_PRIORITIZATION_MATRIX_GUIDE.md)
2. ✅ Score your first project
3. ✅ Complete scoring for all projects
4. ✅ Review and export rankings
5. ✅ Present results to stakeholders

**By End of Week 2**: You should be proficient in portfolio prioritization and management.

---

### Path 3: Administrator (Week 1)

**Day 1** (2 hours):
1. ✅ Read [Administrator Guide](ADMINISTRATOR_GUIDE.md) - Sections 1-3
2. ✅ Create test user accounts
3. ✅ Assign roles and permissions
4. ✅ Review audit logs

**Day 2** (2 hours):
1. ✅ Read Administrator Guide - Sections 4-5
2. ✅ Configure AI providers
3. ✅ Set up one integration
4. ✅ Test connection

**Day 3** (2 hours):
1. ✅ Read Administrator Guide - Sections 6-8
2. ✅ Configure security settings
3. ✅ Set up monitoring and alerts
4. ✅ Create backup

**Day 4-5** (As needed):
1. ✅ Customize templates
2. ✅ Configure prioritization criteria
3. ✅ Train users
4. ✅ Document organization-specific processes

**By End of Week 1**: You should be comfortable with all administrative tasks.

## Frequently Asked Questions

### Which guide should I read first?

- **New to ADPA?** Start with [Getting Started Guide](GETTING_STARTED_GUIDE.md)
- **Need quick lookup?** Use [Quick Reference Guide](QUICK_REFERENCE_GUIDE.md)
- **Portfolio manager?** Read [Portfolio Prioritization Matrix Guide](PORTFOLIO_PRIORITIZATION_MATRIX_GUIDE.md)
- **Administrator?** Go to [Administrator Guide](ADMINISTRATOR_GUIDE.md)

### How long does it take to learn ADPA?

- **Basic proficiency**: 1 day (Getting Started Guide + hands-on practice)
- **Advanced features**: 1-2 weeks (all guides + regular usage)
- **Administrator skills**: 1 week (Administrator Guide + configuration practice)

### Do I need to read all guides?

No. Start with Getting Started Guide, then read guides relevant to your role:

- **All Users**: Getting Started + Quick Reference
- **Project Managers**: Getting Started + Quick Reference + Change Requests Guide
- **Portfolio Managers**: Getting Started + Portfolio Prioritization Matrix
- **Administrators**: Administrator Guide + all other guides (for support)

### Are there video tutorials?

Coming soon! We're producing video tutorials to complement these written guides. Check back for updates.

### Can I print these guides?

Yes! All guides are Markdown format and can be exported to PDF for printing:

1. Open guide in browser
2. Print to PDF (Ctrl/Cmd + P)
3. Or use a Markdown-to-PDF converter

### How do I provide feedback on the guides?

- **Email**: docs@adpa-framework.com
- **In-App**: Use Feedback button
- **Directly**: Contact your administrator who can relay feedback to the product team
